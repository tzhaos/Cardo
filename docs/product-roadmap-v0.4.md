# KhaosBox v0.4 初步产品规划

## 定位

v0.4 的目标不是继续堆更多独立功能，而是把 v0.3 的模板化 box 能力升级为“可选择、可预期、可扩展”的模板体系。用户打开 KhaosBox 时，不再只是看到几个固定 box，而是能按当前任务快速创建合适的工作容器。

## 核心目标

1. 模板选择器成为创建入口
2. 每个模板都有清晰的默认布局、默认字段和默认动作
3. Inbox 到任务容器的分流路径更短
4. Command Center 从导航入口升级为工作入口
5. 为后续自定义模板和模板分享保留模型空间

## 产品原则

模板应该解决具体工作场景，而不是只换一种视觉容器。

每个模板至少回答三个问题：

1. 默认装什么
2. 默认怎么组织
3. 下一步动作是什么

## 候选模板

### Project Board

面向项目推进。默认包含 Backlog、Doing、Review、Done 四列，支持从 Inbox 分流到具体列。

优先级：高

### Reading List

面向链接、文档和待读材料。默认 list 布局，强调标题、来源、状态。

优先级：中

### Resource Hub

面向文件夹、文件、快捷方式和链接。默认 grid 布局，适合做一个主题资源库。

优先级：中

### Daily Desk

面向当天临时工作。默认包含 Inbox 区和 Today 区，适合快速收集、处理和清空。

优先级：中

### Launcher

保留现有启动器模板，强化为快捷入口和常用资源入口。

优先级：已存在，继续打磨

## v0.4 交付切片

### 0.4.1 模板注册表产品化

把当前 Template Registry 扩展为 UI 可读的模板元数据，包括：

1. 模板名称
2. 简短描述
3. 默认布局
4. 默认尺寸
5. 默认状态
6. 创建后的首个推荐动作

### 0.4.2 模板选择器

Command Center 的“新建模板”从简单网格升级为可扫描的模板选择器。每个模板显示名称、图标和一句场景描述。

### 0.4.3 Inbox 分流增强

Inbox 分流目标增加最近使用、推荐目标和搜索过滤。分流后继续保持当前的定位高亮反馈。

### 0.4.4 模板默认内容

允许模板创建时生成默认列、默认 section 或默认说明项。优先服务 Project Board 和 Daily Desk。

### 0.4.5 模板自定义预备

不急着做完整模板编辑器，但数据结构需要为自定义模板预留空间：

1. 模板实例状态
2. 模板定义版本
3. 用户自定义标题和默认布局
4. 未来导入导出兼容策略

## 暂不做

1. 在线模板市场
2. 用户间模板分享
3. 复杂字段系统
4. 自动化规则引擎
5. 多人协作

这些能力会扩大产品面，应该等 v0.4 的本地模板体验稳定后再进入规划。

## 成功标准

1. 用户可以在 5 秒内判断该创建哪个模板
2. 新建模板后不用手动整理基础结构
3. Inbox 中的项目可以快速进入合适的模板位置
4. Command Center 能覆盖创建、搜索、定位三类主路径
5. 新增一个模板不需要改动多个分散模块
